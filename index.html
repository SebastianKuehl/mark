What is mark
md

Local Markdown,
in the browser.

No servers. No build pipeline. No network. Just one command.

mark is a Rust CLI that takes any Markdown file, renders it into a complete self-contained HTML5 document with embedded CSS, and opens it in your default browser — instantly.

When your Markdown links to other .md files, mark follows them recursively, materialises the entire documentation set into a single render run, and rewrites every internal link so navigation works perfectly in the browser.

Every render is stored in ~/.mark/rendered/ under a timestamped directory, isolated and independent. Old runs clean themselves up after 30 days.

PathPurpose
~/.mark/Root app directory
~/.mark/bin/Installed binary

Capabilities

Everything you need,
nothing you don't.

Built with stable Rust and zero network dependencies. Every feature is designed to work offline, stay out of your project directory, and never require configuration to get started.

🔗

Recursive Rendering

Follow local Markdown links transitively. Every file in the linked set is rendered and cross-linked — navigate your entire docs tree without a server.

--recursive
🎨

Themes & Reader Controls

System, light, and dark themes. Persist via config or override per-run. Press t to toggle, c to open the reader config panel.

mark config set-theme
📂

Collapsible Sidebar

Recursive renders include a sidebar tree mirroring your source folder hierarchy. Toggle with e or the sidebar button.

sidebar search
⚡

Render Cache

Unchanged files aren't re-rendered. mark detects when the source is unmodified and opens the existing result instantly.

cache-aware
📄

PDF Export

Export to PDF from the browser UI or via mark pdf using a headless browser. Print styling hides chrome and cleans up colours.

mark pdf
🧘

Zen Mode

Press z to hide all reader chrome and enter a distraction-free reading surface. Press again to return.

keyboard: z
📁

Directory Entry

Run mark inside any docs folder — no file argument needed. mark docs/ works too.

directory-aware
🔒

Fully Self-Contained

No CDN calls. No external fonts at render time. Every rendered HTML file embeds all CSS and JS it needs — works completely offline.

zero network deps
🛠

Shell Completions

Generate and install completions for bash, zsh, fish, and PowerShell with a single command.

mark completions
Installation

Up and running
in two commands.

Homebrew (macOS / Linux)

brew install sebastiankuehl/tap/mark

Installs the latest release via the sebastiankuehl/tap Homebrew tap. The binary is placed on your PATH automatically — no shell restart required.

Linux / macOS

# clone and install
git clone https://github.com/sebastiankuehl/mark
cd mark
bash scripts/install.sh

Builds in release mode, copies the binary to ~/.mark/bin/mark, and adds it to your PATH. Restart your shell or run the printed source command after install.

Windows (PowerShell)

# clone and install
git clone https://github.com/sebastiankuehl/mark
cd mark
.\scripts\install.ps1

No admin required. Installs to %USERPROFILE%\.mark\bin\ and updates your user PATH. Restart PowerShell for the PATH change to take effect.

Usage

Common commands.

Render & Open
$ mark README.md

Renders the file (and any linked Markdown files) and opens the result in your default browser immediately.

Current Directory
$ mark

Discovers all Markdown files in the current directory, renders them together, and opens the first entry point.

Render Without Opening
$ mark -n README.md

Renders and writes the HTML without launching the browser. Useful in CI and headless environments.

Single File Only
$ mark --single guide.md

Renders only the specified file. Local Markdown links are left unchanged and a note is printed for any skipped files.

Set Theme
$ mark config set-theme dark

Persists your theme preference to ~/.mark/config.toml. Supports system, light, and dark.

Export to PDF
$ mark pdf README.md .

Exports directly to PDF using a headless browser CLI if available. Passing . writes the PDF to the current directory.

Clean Up Old Renders
$ mark wipe --old-renders

Removes render run directories older than 30 days from ~/.mark/rendered/ and prints a summary.

Shell Completions
$ mark completions zsh

Generates a shell completion script. Supports bash, zsh, fish, and powershell.
